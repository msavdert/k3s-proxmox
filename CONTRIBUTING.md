# Contributing to K3s Proxmox "Hard Way"

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such an amazing place.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail, including steps to reproduce.
- Mention your Proxmox and Ubuntu versions.

### Suggesting Enhancements
- Open an issue with the tag `enhancement`.
- Explain why the enhancement would be useful.

### Pull Requests
1. Fork the repo.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Commit your changes.
4. Push to the branch.
5. Open a Pull Request.

## Documentation Style
- Keep the "Hard Way" spirit: explain the *why* and the *how*.
- Use clear, professional English.
- Always update the relevant `docs/` chapter.
